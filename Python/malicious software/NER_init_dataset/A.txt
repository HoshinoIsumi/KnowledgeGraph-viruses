用 MBR/引导扇区感染程序感染计算机的唯一方法是尝试从受感染的软盘引导。软盘的引导扇区具有确定软盘是否可引导的代码，并显示“非系统磁盘或磁盘错误”消息。正是该代码隐藏了感染。当出现非系统盘错误消息时，感染已经发生。病毒一旦执行，就会感染硬盘的MBR，并可能驻留在内存中。随后每次启动时，病毒都会被加载到内存中，并尝试感染机器访问的软盘。
Crazy_Boot 病毒于 1995 年 5 月首次在 WildList 上被列为“野生”。自首次报告以来，该病毒在全世界变得非常普遍，但现在很少报告。从受感染的软盘启动后，Crazy_Boot 会将自身安装到内存和硬盘主引导记录中。作为其有效负载，在 2323h (8,995) Int13h 磁盘读取操作之后，Crazy_Boot 在屏幕上（使用 Int10h）显示以下消息：不要用电脑玩！否则你会陷入“深深的”麻烦！疯狂启动版 1.0此时，病毒会进入无限循环，最终要求用户重新启动系统（由于循环太“紧”，Ctrl+Alt+Del 通常不起作用。按重置按钮或重新启动电源通常是必需。）有效负载只能在运行 DOS 或 Windows 95/98 的计算机上激活。在Windows 95/98下，触发它并不以禁用32位驱动程序为条件。它不以 Windows DOS 框中发生的读取操作为条件。即使在使用 Windows 资源管理器时，有效负载也可以激活。然而，在Windows95/98系统上，由于使用了Int10h（向屏幕写入字符），用户将无法看到Carzy_Boot的有效负载消息。计算机屏幕将完全空白，系统将锁定。而在内存中，Crazy_Boot具有读隐形的特性。如果用户尝试查看主引导记录，则主引导记录实际上并不是向用户显示的内容。相反，病毒会读取原始未受感染的主引导记录副本所在的位置，然后显示该副本。Crazy_Boot 将未受感染的主引导记录的副本保存到物理位置柱面 0、扇区、第 4 面。此外，如果用户尝试使用磁盘编辑器程序查看内存中带有病毒的硬盘，病毒将尝试干扰磁盘读取操作，从而产生读取错误。除上述内容外，还应注意以下3点：1）位于0、0、4的原始未感染主引导记录不是以连续格式保存的，因此很难手动将其复制回正确位置。2）新的主引导记录（位于0,0,1）不包含有效的分区表信息。因此，如果用户从干净的软盘启动，他们将暂时无法访问硬盘驱动器。从受感染的系统再次启动将允许用户再次访问硬盘驱动器。3) 干净启动是删除启动病毒的正常部分。对于 VirusScan 等防病毒扫描程序来说，第 2 点没有任何意义 - VirusScan 以不同的方式访问硬盘，因此能够在无法“看到”病毒的情况下删除病毒。Crazy_Boot 感染软盘中的引导扇区和硬盘驱动器上的主引导记录。加载到内存后，该病毒会将 DOS 内存顶部减少 3K。例如，在基于 DOS 的 v6.20 计算机上，[最初有 640K 常规内存可用，并且已经加载了各种其他驱动程序]，内存减少将导致显示 589,120 字节可用，而不是 592,192。请注意，不同的计算机可能会显示不同数量的可用常规内存，具体取决于配置。当启动使用 DOS 操作系统的受感染计算机时，病毒会加载并驻留在内存中。与大多数在 DOS 下运行的内存驻留启动病毒一样，Crazy_Boot 会拦截对 Int13h 的调用，并将调用重定向到其自己的代码。当病毒在内存中时——当它能够拦截对 Int13h 的调用时——几乎任何软盘访问都会导致其感染。当启动运行 Windows 95 操作系统的受感染计算机时，病毒会将自身加载到内存中。然而，由于 Windows 95 使用专门的 32 位文件系统驱动程序，因此病毒无法复制到其他软盘 - 即使在 Windows DOS 环境下也是如此。造成这种情况的原因很简单，就是Windows 95s专用的32位磁盘驱动程序造成的。一旦Windows 95完成启动（更准确地说，一旦VMM32.SYS加载），对Int13h的调用就不会以与DOS相同的方式进行，因此，虽然病毒仍然驻留在硬盘驱动器的主引导记录上，但病毒无法复制。应注意以下几点：1) Windows 95系统感染后，重新启动Windows 95时，用户可能会看到以下性能警告：实际上，该消息表示 Int13h 地址发生更改 - 至少是 Windows 95 记录的最后一次指向的 Int13h 地址。当 Int13h 地址发生更改时，无论是否有更改，都会出现该消息。是否确实对主引导记录进行了物理更改。该消息可能表明存在病毒，但它的出现还有其他有效原因（磁盘压缩软件、磁盘加密软件等）。性能警告消息框只会出现一次。下次系统重新启动时，将不会出现该消息。如果您无意中看到该消息，最好立即调查其背后的原因。2) Windows 95 32位磁盘驱动程序可以被禁用。这样做时，系统被置于“DOS 兼容模式”。一旦进入 DOS 兼容模式，就会调用 Int13h。从理论上讲，这意味着病毒可能再次具有复制能力。就Crazy_Boot 而言，病毒实际上会在DOS 兼容模式下复制（但前提是Windows 95 处于兼容模式下）。
